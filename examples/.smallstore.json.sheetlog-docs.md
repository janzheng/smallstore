# Sheetlog via `.smallstore.json`

Zero-code path: mount a Sheetlog adapter once, start the server, and any agent can read/write the sheet through the MCP tools.

## Setup

1. Deploy the [Sheetlog Apps Script](https://github.com/yawnxyz/sheetlog) and copy the `/exec` URL.
2. Add `SM_SHEET_URL=https://script.google.com/macros/s/.../exec` to `.env`.
3. Copy `examples/.smallstore.json.example` to `./.smallstore.json`.
4. Run `deno task serve`.
5. From an agent:

   ```
   mcp__smallstore__sm_read with collection: "sheets/Sheet1"
   mcp__smallstore__sm_write with collection: "sheets/Sheet1", key: "row-1", data: { ... }
   ```

The `sheets/*` mount routes to Sheetlog. No glue code.

## Contrast: without `.smallstore.json`

```typescript
import { createSmallstore } from '@yawnxyz/smallstore';
import { createSheetlogAdapter } from '@yawnxyz/smallstore/adapters/sheetlog';

const store = createSmallstore({
  adapters: { sheets: createSheetlogAdapter({ sheetUrl: Deno.env.get('SM_SHEET_URL')!, sheet: 'Sheet1' }) },
  defaultAdapter: 'sheets',
});
```

With the config file, the server wires this on boot and MCP surfaces it to agents automatically.

## Appending rows (non-destructive)

`sm_write` / `set()` on sheetlog is **destructive** — it wipes every row in the tab before inserting. For log-shaped collections where you want to append, use `sm_append`:

```
mcp__smallstore__sm_append with collection: "sheets/Sheet1", items: [
  { url: "https://example.com", title: "Example", date: "2026-04-21 10:00:00" }
]
```

Or via HTTP:

```
POST /api/sheets/Sheet1/append
Content-Type: application/json

{ "items": [ { "url": "...", "title": "..." } ] }
```

Wraps Sheetlog's `DYNAMIC_POST` under the hood. If the sheet has an `_id` column and the payload omits it, the Apps Script auto-generates one and returns the assigned id(s) in `_ids: [...]`. Adapters that don't implement `append()` return 501; `sm_append` is currently only on `SheetlogAdapter`.

## Notes

- `sheetUrl` is the Apps Script `/exec` URL, not the sheet edit URL.
- `$SM_SHEET_URL` resolves at server boot — restart after `.env` changes.
- One tab = one collection. `sm_write` overwrites the whole tab — use `sm_append` for log-style writes.
- Rate-limited by Google Sheets API — don't loop `sm_append` without throttling.
